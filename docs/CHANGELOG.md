## [3.4] – 2026-03-28

### Добавлено
- **Модуль безопасного логирования** (`core/logging_filters.py`):
  - `SecretRedactionFilter` — автоматическое скрытие секретов в логах
  - `PIIFilter` — маскировка персональных данных (телефоны, email, карты)
  - 15+ паттернов для детектирования чувствительных данных
  - Функция `setup_secure_logging()` для простой интеграции

- **Валидация критических секретов**:
  - Функция `_validate_critical_secrets()` в `core/config.py`
  - Проверка наличия BOT_TOKEN и PROVIDER_TOKEN
  - Бот не запустится без критических секретов
  - Улучшенная обработка ошибок конфигурации

- **Безопасный шаблон конфигурации**:
  - Файл `.env.example` (178 строк) с подробными комментариями
  - Все секреты заменены на заглушки
  - Команды для генерации новых секретов
  - Security checklist перед запуском
  - Quick start инструкция

### Изменено
- **Устранение хардкода секретов**:
  - `PAYMENT_PROVIDER_TOKEN` теперь загружается только из `.env`

  - Улучшена валидация PROVIDER_TOKEN (warning вместо блокирующей ошибки)

- **Интеграция безопасного логирования**:
  - В `main.py` добавлен вызов `setup_secure_logging()`
  - Все логи автоматически фильтруются через SecretRedactionFilter

### Безопасность

## [3.3] – 2026-03-28

### Удалено
- **Полное удаление голосового синтеза (TTS)**:
  - ❌ Удалён модуль `core/voice_synthesizer.py` (755 строк кода)
  - ❌ Удалена папка `data/voice_cache/` с кэшем аудио
  - ❌ Удалена команда `/voice` и весь голосовой функционал
  - ❌ Удалены зависимости: `torch`, `torchaudio`, `pydub` (~500MB)
  
- **Причины удаления**:
  - Технические ограничения Silero TTS не позволили достичь естественного звучания
  - Невозможность корректной обработки пауз и интонации
  - Голоса звучали неестественно, несмотря на все улучшения
  - Риск введения пользователей в заблуждение (ИИ звучит как человек)
  - Высокое потребление ресурсов и этические соображения

### Изменено
- **Текстовый режим - единственный режим общения**:
  - ✅ Все персонажи (Анна, Артём, Саша) общаются ТОЛЬКО текстом
  - ✅ AI-консультанты работают только через текстовые сообщения
  - ✅ Живые представления персонажей отправляются только текстом
  
- **Улучшена обработка ошибок**:
  - ✅ Добавлена корректная обработка `BadRequest` при парсинге Markdown
  - ✅ Убраны пустые `except Exception: pass` из критических мест
  - ✅ Улучшено логирование реальных ошибок вместо скрытия
  
- **Оптимизация структуры кода**:
  - ✅ Удалено ~830 строк неиспользуемого кода
  - ✅ Упрощена функция `send_persona_intro()` (удалено 20 строк)
  - ✅ Удалена функция `voice_command()` (44 строки)
  - ✅ Удалена отправка голосовых сообщений в `ai_message_handler()` (9 строк)
  - ✅ Удалены неиспользуемые импорты из `handlers/ai_handlers.py`

### Безопасность
- **Конфиденциальность улучшена**:
  - ✅ Удалены все следы обработки биометрических данных (голос)
  - ✅ Уменьшена поверхность атаки (нет TTS = нет связанных уязвимостей)
  - ✅ Снижено потребление ресурсов (нет загрузки torch-моделей)
  
- **Файловая безопасность**:
  - ✅ Удалены опасные файлы: `.bak`, временные логи, тестовые скрипты
  - ✅ Очистка `__pycache__/` перед деплоем
  - ✅ Удалён `info.php` из production
  
- **Конфигурация**:
  - ✅ Продолжена работа по выводу секретов из кода в `.env`
  - ✅ Улучшено экранирование данных в Markdown (`escape_markdown`)
  - ✅ Безопасная обработка callback query (игнорирование устаревших)

### Технические детали
- **Зависимости для ver. 3.2**:
```bash
pip install python-telegram-bot python-dotenv httpx aiohttp requests PyMySQL yookassa cryptography
```

> **Важно:** Зависимости `torch`, `torchaudio`, `pydub` БОЛЬШЕ НЕ ТРЕБУЮТСЯ!

- **Архитектура**:
  - ✅ Чёткое разделение: текстовый режим / AI-консультант / платежи
  - ✅ Упрощение кодовой базы (удалено ~830 строк)
  - ✅ Улучшена читаемость и поддерживаемость

## [3.1] – 2026-03-26

### Добавлено
- **Цветные inline и reply-кнопки**:
  - Навигационные кнопки (`🏠 Главная`, `⬅️ Назад`) теперь имеют синий цвет (`style="primary"`).
  - Кнопки действий (`✅ Оформить заказ`, `🛍️ Продолжить покупки`, `🚀 Да, продолжить!`, `➡️ Продолжить`) — зелёный цвет (`style="success"`).
  - Кнопка `🗑️ Очистить корзину` — красный цвет (`style="danger"`).
  - Кнопка `📦 Каталог` оставлена без стиля, чтобы не смешиваться с навигацией.
- **Поддержка стилей для кнопок из JSON**:
  - В `core/navigation.py` метод `_create_keyboard` теперь считывает поле `Style` из кнопок и передаёт его в `InlineKeyboardButton`.
  - В `data/shop_data.json` добавлено поле `"Style": "primary"` для всех кнопок `🏠 Главная` и `⬅️ Назад`.

### Изменено
- **Методы клавиатур** (`user_keyboards.py`):
  - Добавлены параметры `style` во все методы, где создаются целевые кнопки (`create_back_home`, `create_navigation_only`, `create_quick_actions`, `create_product_actions` и др.).
  - Reply-кнопки (`ReplyKeyboards`) также обновлены с использованием `KeyboardButton(..., style=...)`.
- **Хендлеры**:
  - `handlers/navigation_handlers.py`: в `show_editable_cart` добавлены стили для кнопок корзины.
  - `handlers/payment_handlers.py`: стили для кнопок `🏠 Главная` и `📦 Каталог` (где нужно).
  - `handlers/checkout_handlers.py`: добавлены стили для кнопок оформления заказа (`✅ Продолжить оформление`, `➡️ Продолжить`, `🏠 Главная`, `⬅️ Назад` и т.д.).

### Исправлено
- Устранены пропущенные стили в некоторых callback-обработчиках (например, в `handle_manual_phone_input`, `handle_email_input`, `handle_contact_message`).
- Исправлен синтаксис в `ReplyKeyboards.get_cart_context` (была пропущена кавычка).

### Технические детали
- Требуется версия `python-telegram-bot >= 22.7`.
- Цветные кнопки работают в клиентах Telegram, выпущенных после февраля 2026 года.
